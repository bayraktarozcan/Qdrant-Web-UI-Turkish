# Çok Kiracılı Ortamlarda Kullanıcı Verilerini Ayırma

Bu eğitimde, Qdrant'ta çok kiracılılığın nasıl uygulanacağını ele alacağız. Çok kiracılılık, birden fazla kiracıyı veya istemciyi tek bir Qdrant örneğinde barındırmanıza olanak tanır ve kiracılar arasında veri yalıtımı ile erişim kontrolü sağlar. Bu özellik, farklı müşterilere hizmet verirken verilerinin ayrımını korumanız gereken kullanım durumları için çok önemlidir.

## Adım 1: Koleksiyon oluşturun

Bir öneri hizmeti işlettiğinizi ve farklı departmanların (kiracıların) verilerini Qdrant'ta depoladığınızı hayal edin. **Payload tabanlı çok kiracılılık** kullanarak, tüm kiracıların verilerini tek bir koleksiyonda tutabilir ancak verileri benzersiz bir kiracı tanımlayıcıya göre filtreleyebilirsiniz.

Tüm kiracılar için paylaşılan bir koleksiyon oluşturmak üzere aşağıdaki isteği çalıştırın:

```json withRunButton=true
PUT collections/central_library
{
  "vectors": {
    "size": 4,
    "distance": "Dot"
  }
}
```

## Adım 2: Bir kiracı indeksi oluşturun

Qdrant, çok kiracılı aramaları optimize etmek için kiracının tanımlayıcısına dayalı verimli indekslemeyi destekler. Kiracı indekslemeyi etkinleştirerek, verileri daha hızlı kiracıya özel aramalar için diskte yapılandırabilir, performansı artırabilir ve disk okumalarını azaltabilirsiniz.

Kiracı tanımlayıcısı (`group_id`) için indekslemeyi etkinleştirmek üzere aşağıdaki isteği çalıştırın:

```json withRunButton=true
PUT /collections/central_library/index
{
    "field_name": "group_id",
    "field_schema": {
        "type": "keyword",
        "is_tenant": true
    }
}
```

## Adım 3: Kiracılar için vektörleri yükleyin

Ardından, paylaşılan koleksiyona veri yükleyeceksiniz. Her veri noktası, payload'da kiracıya özgü bir tanımlayıcı ile etiketlenir. Bu tanımlayıcı (`group_id`), aynı koleksiyonda depolandıklarında bile kiracıların verilerinin yalıtılmış kalmasını sağlar.

Veri noktalarını eklemek için aşağıdaki isteği çalıştırın:

```json withRunButton=true
PUT /collections/central_library/points
{
  "points": [
    {
      "id": 1,
      "vector": [0.1, 0.2, 0.3, 0.4],    
      "payload": {
        "group_id": "user_1",
        "station": "Communications",
        "message_log": "Contact with colony headquarters."
      }
    },
    {
      "id": 2,
      "vector": [0.5, 0.6, 0.7, 0.8],
      "payload": {
        "group_id": "user_2",
        "station": "Security",
        "message_log": "Monitor intruder alert system."
      }
    },
    {
      "id": 3,
      "vector": [0.9, 1.0, 1.1, 1.2],
      "payload": {
        "group_id": "user_3",
        "station": "Engineering",
        "message_log": "Repair warp core malfunction."
      }
    }
  ]
}
```

## Adım 4: Filtrelenmiş bir sorgu gerçekleştirin

Paylaşılan koleksiyonu sorgularken, kiracıların yalnızca kendi verilerine erişebilmesini sağlamak için `group_id` payload alanını kullanın. Bu sorgudaki filtre, yalnızca belirtilen `group_id`'ye ait noktaların döndürülmesini sağlar.

`user_1`'e özgü verileri aramak için aşağıdaki isteği çalıştırın:

```json withRunButton=true
POST /collections/central_library/points/query
{
    "query": [0.2, 0.1, 0.9, 0.7],
    "filter": {
        "must": [
            {
                "key": "group_id",
                "match": {
                    "value": "user_1"
                }
            }
        ]
    },
    "limit": 2,
    "with_payload": true
}
```

## Adım 6: Sorgu sonuçlarını gruplayın

Sorgu sonuçlarını `group_id` gibi belirli alanlara göre gruplayarak her kiracının verilerine genel bir bakış elde edebilirsiniz. Bu sorgu, sonuçları `group_id`'ye göre gruplar ve grup sayısı ile grup başına nokta sayısını sınırlar.

Sonuçları `group_id`'ye göre gruplamak için aşağıdaki isteği çalıştırın:

```json withRunButton=true
POST /collections/central_library/points/query/groups
{
    "query": [0.01, 0.45, 0.6, 0.88],
    "group_by": "group_id",  
    "limit": 5,  
    "group_size": 5,
    "with_payload": true  
}
```
