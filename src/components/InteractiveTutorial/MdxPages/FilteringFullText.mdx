# Tam Metin Filtreleme

İşte gezegen verileri ve açıklama alanları içeren bir koleksiyon kullanarak **Qdrant'ta Tam Metin Filtrelemeyi** adım adım anlatan bir eğitim:

## Adım 1: Koleksiyon oluşturun

Önce, boyutu 4 olan vektörler ve benzerlik için dot product mesafesi ile `star_charts` adında bir koleksiyon oluşturuyoruz.

```json withRunButton="true"
PUT /collections/star_charts
{
  "vectors": {
    "size": 4,
    "distance": "Dot"
  }
}
```

## Adım 2: Payload'da açıklamalar içeren veriler ekleyin

Ardından, koleksiyona veri ekliyoruz. Her giriş, bir id, vector ve çeşitli gök cisimleri hakkında koloni bilgisi, yaşam desteği ve açıklama gibi detayları içeren bir payload içerir.

### Adım 3: Tam ifade ile filtrelemeyi deneyin (alt dizin eşleşmesi)

Şimdi, "host colonies" tam ifadesini içeren girişleri bulmak için açıklamaları filtrelemeyi deneyelim.
Qdrant, varsayılan olarak tam eşleşmeler kullanarak metin filtrelemesini destekler, ancak bunun metni tokenize etmeyeceğini unutmayın.

```json withRunButton="true"
POST /collections/star_charts/points/scroll
{
  "filter": {
    "must": [
      {
        "key": "description",
        "match": {
          "text": "host colonies"
        }
      }
    ]
  },
  "limit": 2,
  "with_payload": true
}
```

Bu filtrenin çalıştığını göreceksiniz, ancak ifadeyi biraz değiştirirseniz sonuç döndürmez, çünkü indekslenmemiş metinde alt dizin eşleştirmesi varyasyonlar için yeterince esnek değildir.

**Not**: Kümeniz Qdrant Cloud'da çalışıyorsa bu sorgu başarısız olur. Qdrant Cloud'da koleksiyonlar varsayılan olarak strict modda çalışacak şekilde yapılandırılmıştır, bu da indekslenmemiş alanların sorgulanmasını engeller. Bir alanı metin filtrelemesi için nasıl indeksleyeceğinizi daha sonra öğreneceksiniz.

### Adım 4: Açıklama alanını indeksleyin

Filtrelemeyi daha güçlü ve esnek hale getirmek için `description` alanını indeksleyeceğiz. Bu, metni tokenize ederek "cave colonies" gibi ifadeler için daha karmaşık sorgulara olanak tanır. Bir `word` tokenizer kullanıyoruz ve yalnızca 5 ila 20 karakter arasındaki tokenler indekslenecek.

**Not:** Filtreleme yapmadan önce bir alanı her zaman indekslemelisiniz. İndeks oluşturmadan önce filtreleme kullanırsanız (Adım 3'teki gibi), Qdrant tüm veri kümesinde yapılandırılmamış bir şekilde arama yapar. Arama performansınız çok yavaş olur.

```json withRunButton="true"
PUT /collections/star_charts/index
{
    "field_name": "description",
    "field_schema": {
        "type": "text",
        "tokenizer": "word",
        "lowercase": true
    }
}
```

### Adım 5: Filtreyi tekrar deneyin

İndekslemeden sonra, filtreyi tekrar çalıştırabilirsiniz, ancak bu kez bir ifade aramak yerine.
Şimdi açıklamalardan tüm "caves" VE "colonies" tokenlerini filtreleyeceksiniz.

```json withRunButton="true"
POST /collections/star_charts/points/scroll
{
  "filter": {
    "must": [
      {
        "key": "description",
        "match": {
          "text": "caves colonies"
        }
      }
    ]
  },
  "limit": 2,
  "with_payload": true
}
```
## Özet

İndekslenmemiş bir alanda tam metin filtrelemesi varsayılan olarak alt dizin eşleşmesi yapar veya strict modda çalışan koleksiyonlarda (Qdrant Cloud'da varsayılan) bir hatayla sonuçlanır. Alanı önce indekslemek en iyi uygulamadır. Bu, kelime sırasını tamamen yok sayan sorgular gibi daha gelişmiş tam metin sorgularını mümkün kılar.
