# Seyrek Vektör Arama

Bu eğitimde, Qdrant'ta seyrek vektörlerle bir koleksiyon oluşturmayı, seyrek vektörlerle noktalar eklemeyi ve belirli indekslere ve değerlere dayalı olarak sorgulamayı öğreneceksiniz. Seyrek vektörler, yalnızca belirli boyutların sıfır olmadığı verileri verimli bir şekilde depolamanıza ve aramanıza olanak tanır; bu, özellikle metin yerleştirmeleri veya seyrek verilerin işlenmesi gibi uygulamalarda kullanışlıdır.

## Adım 1: Seyrek vektörlerle koleksiyon oluşturun

İlk adım, seyrek vektörleri işleyebilen bir koleksiyon oluşturmaktır. Tam özellik uzaylarını temsil eden yoğun vektörlerin aksine, seyrek vektörler yalnızca seçili konumlardaki sıfır olmayan değerleri depolar ve bu da onları daha verimli kılar. Her noktanın anahtar kelimeleri veya diğer özellikleri temsil etmek için seyrek vektörlere sahip olacağı `sparse_charts` adında bir koleksiyon oluşturacağız.

Koleksiyonu oluşturmak için aşağıdaki isteği çalıştırın:

```json withRunButton=true
PUT /collections/sparse_charts
{
    "sparse_vectors": {
        "keywords": {}
    }
}
```

### Açıklama:
- **`sparse_vectors`**: Koleksiyonun seyrek vektörleri desteklediğini tanımlar; bu durumda "keywords" ile indekslenir. Bu, yalnızca belirli indekslerin (konumların) sıfır olmayan değerlere sahip olduğu anahtar kelime tabanlı özellikleri temsil edebilir.

---

## Adım 2: Seyrek vektörlerle veri noktaları ekleyin

Koleksiyon hazır olduğunda, seyrek vektörlerle noktalar ekleyebilirsiniz. Her nokta şunları içerecektir:
- `indices`: Vektör uzayındaki sıfır olmayan değerlerin konumları.
- `values`: Bu konumlardaki karşılık gelen değerler; her bir anahtar kelimenin veya özelliğin önemini veya ağırlığını temsil eder.

Noktaları eklemek için aşağıdaki isteği çalıştırın:

```json withRunButton=true
PUT /collections/sparse_charts/points
{
    "points": [
        {
            "id": 1,
            "vector": {
                "keywords": {
                    "indices": [1, 42],
                    "values": [0.22, 0.8]
                }
            }
        },
        {
            "id": 2,
            "vector": {
                "keywords": {
                    "indices": [2, 35],
                    "values": [0.15, 0.65]
                }
            }
        },
        {
            "id": 3,
            "vector": {
                "keywords": {
                    "indices": [10, 42],
                    "values": [0.3, 0.5]
                }
            }
        },
        {
            "id": 4,
            "vector": {
                "keywords": {
                    "indices": [0, 3],
                    "values": [0.4, 0.3]
                }
            }
        },
        {
            "id": 5,
            "vector": {
                "keywords": {
                    "indices": [2, 4],
                    "values": [0.9, 0.8]
                }
            }
        }
    ]
}
```

### Açıklama:
- Her nokta, belirli anahtar kelime indeksleri ve değerleriyle tanımlanan seyrek vektörü ile temsil edilir.
- Örneğin, **Nokta 1**, sırasıyla `1` ve `42` konumlarında `0.22` ve `0.8` seyrek vektör değerlerine sahiptir. Bunlar, bu konumlarla ilişkili anahtar kelimelerin göreceli önemini temsil edebilir.

---

## Belirli indeksler ve değerlerle sorgu çalıştırma

Bu sorgu, `[1, 42]` konumlarında sıfır olmayan değerlere ve `[0.22, 0.8]` belirli değerlerine sahip noktaları arar. Bu hedefli bir sorgudur ve bu indekslere ve değerlere yakın bir eşleşme bekler.

```json withRunButton=true
POST /collections/sparse_charts/points/query
{
    "query": {
        "indices": [1, 42],
        "values": [0.22, 0.8]
    },
    "using": "keywords"
}
```

**Beklenen sonuç:** **Nokta 1** en iyi eşleşme olacaktır çünkü seyrek vektörü, benzerlik ölçüsünü maksimize eden bu indeksleri içerir. Bu durumda bu, dot product hesaplamasıdır.

---

## Puanlama mekanizmasını anlama

Bu sorgu, `[0, 2, 4]` konumlarında sıfır olmayan değerlere ve `[0.4, 0.9, 0.8]` değerlerine sahip noktaları arar. Örtüşen indeksler ve benzer değerlerle birden fazla eşleşme döndürebilecek daha geniş bir aramadır.

```json withRunButton=true
POST /collections/sparse_charts/points/query
{
    "query": {
        "indices": [0, 2, 4],
        "values": [0.4, 0.9, 0.8]
    },
    "using": "keywords"
}
```
---

## Bu sonucu nasıl elde ettik:

**Nokta 4** ve **Nokta 5**'in seyrek vektörlerinin şöyle olduğunu varsayalım:
- **Nokta 4**: `[0.4, 0, 0, 0, 0]`  (Sorguyla indeks 0'da 0.4 değeriyle eşleşir)
- **Nokta 5**: `[0, 0, 0.9, 0, 0.8]`  (Sorguyla indeks 2 ve 4'te 0.9 ve 0.8 değerleriyle eşleşir)

Dot product şöyle görünür:
- **Nokta 4 için dot product**: 
   - Sorgu: `[0.4, 0, 0.9, 0, 0.8]`
   - Nokta 4: `[0.4, 0, 0, 0, 0]`
   - Dot product: ( 0.4 * 0.4 = 0.16 \)

- **Nokta 5 için dot product**:
   - Sorgu: `[0.4, 0, 0.9, 0, 0.8]`
   - Nokta 5: `[0, 0, 0.9, 0, 0.8]`
   - Dot product: ( 0.9 * 0.9 + 0.8 * 0.8 = 0.81 + 0.64 = 1.45 )

**Nokta 5** daha yüksek bir dot product puanına sahip olduğu için **Nokta 4**'ten daha iyi bir eşleşme olarak kabul edilir.
