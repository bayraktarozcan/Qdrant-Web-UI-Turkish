# Gelişmiş Filtreleme - İç İçe Filtreler

## Adım 1: Koleksiyon Oluşturun

Vektör boyutu 4 ve mesafe ölçütü `Dot` olan `dinosaurs` adında bir koleksiyon oluşturarak başlayın:

```json withRunButton="true"
PUT collections/dinosaurs
{
  "vectors": {
    "size": 4,
    "distance": "Dot"
  }
}
```

## Adım 2: Payload İçeren Vektörler Ekleyin

Şimdi koleksiyona noktalar ekleyebilirsiniz. Her nokta bir `id`, `vector` ve dinozor türü ile beslenme tercihleri gibi ek bilgiler içeren bir `payload` içerir. Örneğin:

```json withRunButton="true"
PUT collections/dinosaurs/points
{
  "points": [
    {
      "id": 1,
      "vector": [0.1, 0.2, 0.3, 0.4],
      "payload": {
        "dinosaur": "t-rex",
        "diet": [
          { "food": "leaves", "likes": false },
          { "food": "meat", "likes": true }
        ]
      }
    },
    {
      "id": 2,
      "vector": [0.2, 0.3, 0.4, 0.5],
      "payload": {
        "dinosaur": "diplodocus",
        "diet": [
          { "food": "leaves", "likes": true },
          { "food": "meat", "likes": false }
        ]
      }
    }
  ]
}
```


## Adım 3: Filtrelemeden önce alanları indeksleyin

**Not:** Filtreleme yapmadan önce bir alanı her zaman indekslemelisiniz. İndeks oluşturmadan önce filtreleme kullanırsanız, Qdrant tüm veri kümesinde yapılandırılmamış bir şekilde arama yapar. Arama performansınız çok yavaş olur.

```json withRunButton="true"
PUT /collections/dinosaurs/index
{
    "field_name": "diet[].food",
    "field_schema": "keyword"
}
```

```json withRunButton="true"
PUT /collections/dinosaurs/index
{
    "field_name": "diet[].likes",
    "field_schema": "bool"
}
```

## Adım 4: `match` ile Temel Filtreleme

Noktaları belirli payload değerlerine göre filtreleyebilirsiniz. Örneğin, aşağıdaki sorgu şu koşulları sağlayan noktaları eşleştirir:

- `diet[].food` "meat" içeriyor.
- `diet[].likes` `true` olarak ayarlanmış.

Her iki nokta da bu koşulları karşılar:
- "t-rex" et yer ve sever.
- "diplodocus" et yer ama sevmez.

```json withRunButton="true"
POST /collections/dinosaurs/points/scroll
{
  "filter": {
    "must": [
      {
        "key": "diet[].food",
        "match": {
          "value": "meat"
        }
      },
      {
        "key": "diet[].likes",
        "match": {
          "value": true
        }
      }
    ]
  }
}
```

Ancak, yalnızca her iki koşulun da dizi içindeki aynı öğe için geçerli olduğu noktaları (örneğin, ID 1 olan "t-rex") almak istiyorsanız, **iç içe bir filtre** kullanmanız gerekir.

## Adım 5: İç İçe Nesne Filtreleri ile Gelişmiş Filtreleme

Filtreyi dizi öğesi düzeyinde uygulamak için `nested` filtre koşulunu kullanırsınız. Bu, `food` ve `likes` değerlerinin her bir dizi öğesi içinde birlikte değerlendirilmesini sağlar:

```json withRunButton="true"
POST /collections/dinosaurs/points/scroll
{
  "filter": {
    "must": [
      {
        "nested": {
          "key": "diet",
          "filter": {
            "must": [
              {
                "key": "food",
                "match": {
                  "value": "meat"
                }
              },
              {
                "key": "likes",
                "match": {
                  "value": true
                }
              }
            ]
          }
        }
      }
    ]
  }
}
```

Bu filtre ile yalnızca "t-rex" (ID 1) döndürülür, çünkü onun dizi öğesi her iki koşulu da karşılar.

### Açıklama

İç içe filtreler, her bir dizi öğesini ayrı bir nesne olarak ele alır ve filtreyi her öğeye bağımsız olarak uygular. Ana belge (bu durumda dinozor noktası), herhangi bir dizi öğesi tüm koşulları karşılarsa filtreyle eşleşir.

## Adım 6: `has_id` ile İç İçe Filtreleri Birleştirme

`has_id`'nin iç içe bir filtrenin içinde kullanılamayacağını unutmayın. Ayrıca ID'ye göre filtreleme yapmanız gerekiyorsa, `has_id` koşulunu ayrı bir cümle olarak ekleyin:

Farklı bir sonuç almayacaksınız. Bu filtrenin, iç içe diet filtresini açık bir ID eşleşmesiyle birleştirerek "t-rex" (ID 1) ile eşleştiğini görebilirsiniz.

```json withRunButton="true"
POST /collections/dinosaurs/points/scroll
{
  "filter": {
    "must": [
      {
        "nested": {
          "key": "diet",
          "filter": {
            "must": [
              {
                "key": "food",
                "match": {
                  "value": "meat"
                }
              },
              {
                "key": "likes",
                "match": {
                  "value": true
                }
              }
            ]
          }
        }
      },
      {
        "has_id": [1]
      }
    ]
  }
}
```
